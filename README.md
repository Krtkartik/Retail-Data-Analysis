# Retail-Data-Analysis

Project Summary: Retail Data Analysis
🧾 Overview

This project focuses on analyzing retail sales data to identify trends, patterns, and business insights. The dataset consists of store-level weekly sales along with additional features such as holidays and store types.

📂 Dataset Used

The analysis is based on three datasets:

Sales Data – Contains weekly sales for each store and department
Features Data – Includes external factors like temperature, fuel price, and holidays
Stores Data – Contains store type and size information

These datasets were merged to perform a comprehensive analysis.

⚙️ Data Preparation
Converted date column into proper date format
Cleaned and handled data inconsistencies
Created new columns such as:
Month
Month Number (for sorting)

📊 Analysis Performed
Sales Trend Analysis – Studied how sales change over time
Store Type Analysis – Compared performance of different store types
Holiday Impact Analysis – Evaluated how holidays affect sales
Monthly Analysis – Identified seasonal patterns in sales

📈 Key Insights
Sales show clear fluctuations over time indicating seasonal trends
Certain store types (e.g., Type A) perform better than others
Sales are generally higher during holiday periods
Some months consistently generate higher revenue

💡 Business Recommendations
Increase inventory during high-demand months
Focus on high-performing store types
Plan promotions and marketing campaigns during holidays
Use data-driven strategies for better decision-making

🏁 Conclusion

The project demonstrates how data analysis can be used to extract meaningful insights from retail data. The dashboard built using Microsoft Power BI helps in visualizing trends and supports better business decisions.
